Alright — this error is actually a very common (but annoying) React Native + Expo issue, especially in release builds. Let’s break it down properly 👇

❌ Your Main Error
TypeError: Cannot read property 'useState' of null

This means:
👉 React is coming as null somewhere
👉 So hooks like useState are breaking

💥 Root Causes (Most Likely in Your Case)

From your logs + setup, one of these is happening:

1. 🚨 Multiple React Versions (VERY COMMON)

You already said:

"i have 2 node_modules"

That’s a big red flag.

👉 If you have:

one node_modules in root

one inside /mobile

Then React might be duplicated → app crashes in release

2. 🚨 Metro / Bundle mismatch (Expo issue)

Expo sometimes bundles incorrectly in release if cache is dirty.

3. 🚨 Wrong import (less likely but check)

Somewhere you might have:

import React, { useState } from 'react';

But React is resolving incorrectly.

4. 🚨 Expo Router / Navigation conflict

Your stack shows:

EnsureSingleNavigator
NavigationContainerInner
ExpoRoot

👉 Could be:

multiple NavigationContainers

or incorrect layout setup

✅ FIX — Do This STEP BY STEP (Don’t skip)
🧹 Step 1: Delete ALL node_modules

From BOTH folders:

root/
mobile/

Then run:

rm -rf node_modules
rm -rf mobile/node_modules
rm -rf package-lock.json
rm -rf mobile/package-lock.json
📦 Step 2: Install clean dependencies

Go to your main project folder:

npm install

If mobile is separate:

cd mobile
npm install
🔍 Step 3: Ensure SINGLE React version

Run this:

npm ls react

👉 You should see ONLY ONE version

🔥 Step 4: Clear Expo cache (IMPORTANT)
npx expo start -c
🏗 Step 5: Rebuild APK properly

If using local build:

npx expo prebuild
npx expo run:android

cd android
./gradlew clean
